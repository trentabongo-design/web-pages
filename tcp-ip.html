Transmission Control Protocol / Internet Protocol (TCP/IP)

Protocols — Term 2

What Is TCP/IP?

TCP/IP stands for Transmission Control Protocol / Internet Protocol. It is the set of communication rules that governs how data is sent and received across the internet. Essentially, it is the foundation that makes it possible for different computers and networks around the world to talk to each other, regardless of what hardware or operating system they use.

The name refers to two separate but closely related protocols that work together. TCP handles how data is broken up, sent, and reassembled, while IP handles the addressing — making sure the data reaches the right destination.

How TCP Works

When you send data over the internet — say, loading a webpage — TCP breaks that data into smaller pieces called packets. Each packet is sent independently across the network and may even take a different route to the destination. When all the packets arrive, TCP reassembles them in the correct order.

TCP also includes error-checking. If a packet gets lost or corrupted along the way, TCP detects this and requests that the packet be resent. This makes TCP reliable — you can be confident that the data you receive is complete and correct.

How IP Works

While TCP deals with the reliability of delivery, IP deals with addressing and routing. Every device connected to the internet is assigned an IP address — a unique numerical label (e.g. 192.168.1.1) that identifies it on the network.

When data packets travel across the internet, IP makes sure each one is labelled with the correct source and destination address, so network routers know exactly where to send it. This is similar to writing a postal address on an envelope before sending it in the mail.

Key Facts About TCP/IP

  • TCP/IP is the core communication protocol suite of the internet.
  • TCP stands for Transmission Control Protocol — it handles reliability and ordering of data.
  • IP stands for Internet Protocol — it handles addressing and routing.
  • Data is split into packets before being sent and reassembled at the destination.
  • Every device on the internet has a unique IP address.
  • TCP/IP was developed in the 1970s by Vint Cerf and Bob Kahn.
  • IPv4 uses 32-bit addresses; IPv6 uses 128-bit addresses to allow far more devices.
